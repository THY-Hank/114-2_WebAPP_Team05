# Team05HW 環境建立教程（給第一次接觸 Docker/K8s）

這份文件的目標是讓你在 Windows 或 macOS（Apple Silicon / Intel）電腦上，從零到可以打開專案畫面。

你不需要先懂 Docker 或 Kubernetes，只要照步驟做。

---

## 1. 你會完成什麼

完成後你可以：

1. 用 Docker 建出專案需要的映像
2. 用 Kubernetes 在本機啟動前後端與資料庫
3. 用瀏覽器開啟專案頁面
4. 用腳本快速啟動、查看狀態、關閉

---

## 2. 事前準備

請先安裝：

1. Git
2. Docker Desktop（Windows 或 macOS）
3. VS Code（建議）

OS 建議：

1. Windows：建議搭配 Git Bash 或 WSL2 終端執行指令。
2. macOS：使用 Terminal 或 iTerm2，並確認已安裝 Rosetta（Apple Silicon 僅在需要 x86 工具時）。

安裝 Docker Desktop 後，請確認：

1. Docker Desktop 可以正常開啟
2. Docker Engine 有啟動（不是停止狀態）

---

## 3. 啟用 Docker Desktop 的 Kubernetes

在 Docker Desktop 設定內：

1. 打開 Settings
2. 找到 Kubernetes
3. 勾選 Enable Kubernetes
4. 按 Apply & Restart

等待狀態變成 Running。

可用下列指令確認：

```bash
kubectl config current-context
kubectl get nodes
```

如果成功，通常 context 會是 `docker-desktop`。

---

## 4. 下載專案與切到根目錄

```bash
git clone <你的repo網址>
cd Team05HW
```

如果你已經有專案資料夾，就直接進到專案根目錄即可。

Windows / macOS 指令完全相同。

---

## 5. 準備本機秘密設定（不會上傳 GitHub）

我們使用本機檔案保存敏感資訊，避免外洩。

1. 複製範本檔：

```bash
cp .env.k8s.dev.example .env.k8s.dev
```

2. 打開 `.env.k8s.dev`，填入你自己的值：

- SECRET_KEY
- DB_PASSWORD
- JWT_SECRET
- AES_KEY
- AES_IV

說明：

1. `.env.k8s.dev` 會被 git 忽略，不會被 push
2. `.env.k8s.dev.example` 是可提交的範本檔

---

## 6. 先建映像（第一次需要）

```bash
docker compose build backend frontend
```

如果你之前已經 build 過，這步可略過。

若你用的是 Apple Silicon（M1/M2/M3），這個專案的 Docker 映像可直接建置，不需要額外改平台參數。

---

## 7. 啟動本機 K8s 開發環境（推薦）

直接執行一鍵腳本：

```bash
bash scripts/k8s-dev-up.sh
```

這個腳本會自動做：

1. 檢查 Kubernetes 連線
2. 建立 namespace
3. 從 `.env.k8s.dev` 建立/更新 secret
4. 套用 dev overlay
5. 檢查 rollout

---

## 8. 打開專案頁面

最穩定方式是 port-forward：

```bash
kubectl port-forward svc/frontend 8080:80 -n team05
```

再打開瀏覽器：

http://localhost:8080

關閉 port-forward：

1. 在該終端按 Ctrl+C
2. 只會停止轉發，不會刪除服務

---

## 9. 常用指令（請先記這幾個）

查看狀態：

```bash
bash scripts/k8s-dev-status.sh
```

關閉 dev 資源：

```bash
bash scripts/k8s-dev-down.sh
```

關閉並連 secret 一起刪除：

```bash
bash scripts/k8s-dev-down.sh --purge-secret
```

## 10. 常見錯誤與解法

### A. Docker 連不上（pipe/dockerDesktopLinuxEngine）

現象：

- docker 指令報錯
- 類似找不到 dockerDesktopLinuxEngine

解法：

1. 先開 Docker Desktop
2. 等待引擎啟動完成
3. 再執行命令

這個錯誤主要是 Windows 常見訊息。

macOS 常見的是 daemon 未啟動，通常會看到類似 `Cannot connect to the Docker daemon`，做法一樣：先確認 Docker Desktop 已完全啟動。

---

### B. ImagePullBackOff

現象：

- Pod 卡在 ImagePullBackOff

常見原因：

1. 本機沒 build image
2. 用到雲端映像名稱但本機沒有權限拉取

解法：

1. 先執行 `docker compose build backend frontend`
2. 再跑 `bash scripts/k8s-dev-up.sh`

---

### C. 腳本說缺少 SECRET_KEY 或 DB_PASSWORD

現象：

- `k8s-dev-up.sh` 直接中止

解法：

1. 檢查 `.env.k8s.dev` 是否存在
2. 檢查檔案內 5 個 key 是否都填了

---

### D. 網頁打不開 localhost:8080

解法：

1. 確認 port-forward 還在執行中
2. 另一個終端執行 `bash scripts/k8s-dev-status.sh` 看 frontend pod 是否 Running

macOS 若遇到埠占用，可改用：

```bash
kubectl port-forward svc/frontend 8081:80 -n team05
```

然後改開 `http://localhost:8081`。

---

## 11. 我們專案有兩條路，請不要混淆

### 本機開發路線（你現在走這條）

1. 不需要 GitHub Secrets
2. 不需要 push 才能啟動
3. 主要使用 `k8s/overlays/dev`

### 自動部署路線（CI/CD）

1. push 到 main/master 才會觸發
2. 需要設定 GitHub repository secrets
3. 主要使用 `k8s/overlays/prod`

---

## 12. 給組員的最短上手清單

如果你只想最快跑起來，做這 5 步：

1. 開 Docker Desktop，確認 Kubernetes 是 Running
2. `cp .env.k8s.dev.example .env.k8s.dev`
3. 編輯 `.env.k8s.dev`，填入 5 個秘密值
4. `docker compose build backend frontend`
5. `bash scripts/k8s-dev-up.sh`，再 `kubectl port-forward svc/frontend 8080:80 -n team05`

完成。

---

## 13. Windows / macOS 快速對照

1. Docker Desktop 安裝：兩者都用官方 Docker Desktop。
2. Kubernetes 啟用：兩者都在 Docker Desktop Settings 內啟用。
3. 指令差異：本文件所有核心指令在 Windows（Git Bash/WSL）與 macOS 都一致。
4. 終端建議：
	- Windows：Git Bash 或 WSL2
	- macOS：Terminal 或 iTerm2
